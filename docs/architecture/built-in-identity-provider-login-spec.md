# 生产可用的内建身份提供方与登录模块规格

- 日期：2026-06-07
- 相关任务：T011
- 相关模块：M01、M02、M11
- 目标阶段：P1 只读诊断 MVP
- 状态：已确认，首批验收范围已落地

> 2026-06-08 实现状态补充：正式内建身份模式下的账号、密码、会话、管理员重置密码、首次改密、会话撤销、浏览器登录接线和相关自动化测试已完成；`local-oidc` 继续仅作为本地联调能力保留。

## 1. 背景

当前控制面已经具备以下能力：

- 开发态 `dev-hs256` Bearer Token 校验；
- 真实 OIDC / 外部 IdP 接入模式；
- 基于 Spring Security 的标准浏览器登录入口、会话查询和退出入口；
- 服务端基础 RBAC、结构化授权拒绝和追加式审计记录；
- 本地 `local-oidc` Mock OIDC 联调能力。

这些能力足以支撑“接入已有企业 IdP”的场景，也足以支撑开发与本地联调，但在本规格编写时仍不等于“生产可用的内建身份模块”。本规格定义的首批验收范围现已完成落地，后续扩展项仅保留在完整 MFA、自助找回密码和内建 OIDC 对外发行等后续阶段。

本规格用于定义“在生产环境没有企业 SSO / 外部 IdP 的前提下，系统内建正式身份能力”的范围、边界和实施约束。

## 2. 目标与非目标

### 2.1 目标

- 为单组织部署场景提供正式可用的内建身份提供方与浏览器登录能力。
- 在不降低现有安全基线的前提下，为控制面建立正式账号、密码、会话与审计闭环。
- 保持 `M01 -> M02` 的职责边界清晰，让身份认证与策略授权可以独立演进。
- 保持与未来外部 IdP 接入的统一身份模型兼容，避免后续再次重构授权链路。

### 2.2 非目标

- 不把 `local-oidc` / Mock OIDC 升格为生产方案。
- 不引入多租户、租户标识、租户隔离或租户路由。
- 不在本轮实现完整 MFA，只保留扩展口。
- 不在本轮实现邮件自助找回密码、短信验证码或外部通知通道。
- 不在本轮引入生产写执行、审批绕过或与 P1 无关的平台能力。

## 3. 现状与差距

### 3.1 当前属于 M01 的能力

- Bearer Token 校验与 OIDC/JWT Claim 映射；
- 浏览器登录入口、会话查询和退出入口；
- `OperatorIdentity` 统一身份模型与角色前缀标准化；
- 本地联调专用 `local-oidc` Mock Provider。

### 3.2 当前属于 M02 的能力

- 基于动作到角色映射的基础 RBAC；
- `/internal/**` 统一认证复用、授权拒绝和执行上下文注入；
- 追加式审计记录与最新审计查询。

### 3.3 当前主要差距

必须项：

- 正式账号模型与持久化事实源；
- 正式密码凭据模型、强哈希和校验策略；
- 浏览器会话持久化、撤销和生命周期治理；
- 账号锁定、禁用、管理员受控重置密码、首次登录改密；
- 与 `M02` 衔接的角色来源与身份状态失效机制；
- 身份与会话审计事件；
- 生产模式与本地联调模式的显式隔离；
- 相关版本化契约与测试门禁。

增强项：

- MFA 实际验证器；
- 邮件自助找回密码；
- 记住设备、风险感知登录、CAPTCHA；
- 运维管理界面。

## 4. 设计原则

- 所有登录输入、密码、令牌、会话标识和浏览器请求都视为不可信输入。
- 身份认证属于 `M01`；策略决策与审计属于 `M02`；两者不得互相越权。
- 关系型数据库是正式身份与会话事实源；Redis 仅可用于缓存、锁、限流和短期辅助状态。
- 浏览器不得保存授权事实，只能保存随机会话标识或短期挑战标识。
- 账号状态变化、角色变化和密码重置必须能触发会话失效。
- 所有生产密钥必须来自受控密钥源或受管配置，不得写死在仓库、测试数据或示例生产配置中。

## 5. 模块边界

### 5.1 M01 负责的能力

- 账号模型与账号状态管理；
- 密码凭据存储、哈希校验、参数升级；
- 登录入口、登录校验、登录失败计数与锁定；
- 浏览器会话创建、查询、续期、失效与撤销；
- 管理员受控重置密码、首次登录改密；
- 统一 `OperatorIdentity` 生成；
- 内建 OIDC 发行能力与其浏览器登录集成；
- 身份相关审计事件的业务发起。

### 5.2 M02 负责的能力

- 对 `OperatorIdentity` 做策略决策；
- 维护动作到角色或后续策略源的映射；
- 记录授权决策、拒绝原因和关联审计；
- 定义安全拒绝的结构化错误；
- 将身份状态带来的访问后果转化为策略和审计结果。

### 5.3 bootstrap 只负责的内容

- Spring Security / WebFlux 过滤链装配；
- 将 HTTP 请求转成 `M01` 应用命令或查询；
- 将 `M01` 产生的身份接入 `M02` 鉴权链路；
- 装配运行模式和 profile；
- 注册控制器、过滤器、异常映射和基础配置。

### 5.4 明确禁止的做法

- `bootstrap` 直接读账号表或会话表实现业务逻辑；
- `M02` 直接读 `M01` 持久化表推导角色；
- 前端根据展示文本推断是否已授权；
- 用 `local-oidc` 代码路径承接生产登录。

## 6. 运行模式

### 6.1 开发模式

- 可继续保留 `dev-hs256`；
- 用于后端开发、自动化测试与最小化调试；
- 明确禁止部署到生产。

### 6.2 本地联调模式

- 使用 `local-oidc` profile；
- 保留 Mock OIDC 闭环；
- 仅用于本地前端和控制面联调；
- 不作为测试环境或生产环境方案。

### 6.3 生产内建身份模式

- 启用正式账号、密码和会话事实源；
- 启用正式登录页和正式会话治理；
- 启用正式密钥配置和审计保留策略；
- 禁止启用 `local-oidc`。

### 6.4 生产外部 IdP 模式

- 未来接入外部 IdP 时仍然复用统一 `OperatorIdentity`；
- 与本规格兼容，但不在本轮实施范围。

## 7. 账号模型

建议在 `M01` 中定义以下核心聚合和只读视图。

### 7.1 Account

字段建议：

- `accountId`
- `username`
- `displayName`
- `email`
- `status`
- `passwordState`
- `mfaRequirement`
- `failedLoginCount`
- `lockedUntil`
- `disabledReason`
- `lastLoginAt`
- `createdAt`
- `updatedAt`

约束：

- `username` 全局唯一，大小写归一规则必须固定；
- 单组织部署，不包含任何租户字段；
- `status` 至少包含 `ACTIVE`、`LOCKED`、`DISABLED`、`PASSWORD_RESET_REQUIRED`。

### 7.2 AccountRoleGrant

字段建议：

- `grantId`
- `accountId`
- `roleCode`
- `source`
- `effectiveFrom`
- `effectiveTo`
- `createdBy`
- `createdAt`
- `revokedBy`
- `revokedAt`

角色授予是 `M01` 的事实源，`M02` 只消费结果。

### 7.3 PasswordCredential

字段建议：

- `credentialId`
- `accountId`
- `hashAlgorithm`
- `hashParameters`
- `passwordHash`
- `passwordVersion`
- `mustChangeOnNextLogin`
- `createdAt`
- `rotatedAt`

要求：

- 仅保存不可逆哈希；
- 哈希算法与参数需要显式落库，支持后续升级；
- 至少记录最近一次历史摘要，阻止立即复用旧密码。

### 7.4 AccountSession

字段建议：

- `sessionId`
- `accountId`
- `sessionState`
- `issuedAt`
- `lastSeenAt`
- `expiresAt`
- `absoluteExpiresAt`
- `revokedAt`
- `revokedReason`
- `clientIpHash`
- `userAgentHash`

会话事实源必须可查询、可撤销、可审计。

### 7.5 PasswordResetTicket

字段建议：

- `ticketId`
- `accountId`
- `ticketState`
- `issuedBy`
- `issuedAt`
- `expiresAt`
- `consumedAt`
- `reason`

本轮只支持管理员受控发起。

## 8. 登录流程

### 8.1 浏览器登录主流程

1. 浏览器访问 `/auth/login`。
2. `bootstrap` 将请求路由到 `M01` 登录入口。
3. `M01` 根据运行模式：
   - 生产内建身份模式：返回正式登录页或提交入口；
   - 生产外部 IdP 模式：重定向到外部 OIDC；
   - 本地联调模式：走 `local-oidc`。
4. 用户提交用户名和密码。
5. `M01` 校验账号状态、密码哈希、锁定状态和重置要求。
6. 校验成功后创建服务端会话，并返回安全 Cookie。
7. `PolicyEnforcementWebFilter` 通过 `M01` 解析当前会话，得到统一 `OperatorIdentity`。
8. 后续 `/internal/**` 继续复用现有 `M02` 授权与审计链路。

### 8.2 失败与锁定流程

- 密码错误时增加失败计数；
- 达到阈值后账号进入 `LOCKED`；
- 锁定到期前拒绝再次登录；
- 管理员可以手动解除锁定；
- 失败原因向客户端只返回结构化错误码，不返回可枚举用户信息。

### 8.3 首次登录改密流程

- 管理员重置密码后账号进入 `PASSWORD_RESET_REQUIRED`；
- 用户使用临时凭据登录成功后，不直接授予完整会话；
- 必须先完成改密，再建立正式会话；
- 改密成功后清除重置态，刷新凭据版本并作废旧会话。

## 9. 密码存储与校验策略

- 使用强密码哈希算法，要求可配置成本参数并支持未来升级。
- 哈希过程必须包含独立随机盐。
- 生产密码策略至少包含：
  - 最小长度；
  - 常见弱密码拒绝；
  - 新密码与当前密码不同；
  - 最近一次密码不得复用。
- 校验逻辑必须避免基于响应差异泄露账号是否存在。
- 密码、重置票据和原始登录输入均不得写入日志、审计正文或提示文本。

## 10. MFA 扩展口

本轮不实现完整 MFA，只定义扩展点：

- `mfaRequirement`：账号是否要求二次认证；
- `SecondFactorChallenge`：预留挑战模型；
- `SECOND_FACTOR_REQUIRED`：登录后待完成二次认证状态；
- `SecondFactorVerifier`：预留接口。

要求：

- 默认关闭；
- 不影响当前单因素登录闭环；
- 后续扩展时不破坏会话和审计模型。

## 11. 浏览器会话模型

- 采用服务端持久化会话；
- 浏览器 Cookie 只存随机会话标识；
- Cookie 必须启用 `HttpOnly`；
- 生产必须启用 `Secure`；
- `SameSite` 至少支持 `Lax`，必要时可配置；
- 登录成功后必须更换会话标识，防止会话固定攻击；
- 空闲超时与绝对超时必须同时存在；
- 账号禁用、密码重置、角色撤销和管理员撤销都能让会话失效。

## 12. Token / Session 生命周期

### 12.1 浏览器会话

- 空闲超时：例如 15 到 30 分钟，具体值由配置决定；
- 绝对过期：例如 8 到 12 小时，具体值由配置决定；
- 关键状态变化后立即失效。

### 12.2 内建 OIDC Token

如果生产内建身份模式需要向内部组件发放 OIDC/JWT，则：

- Access Token 必须短期有效；
- 签名密钥必须可轮换；
- 至少支持 `iss`、`sub`、`aud`、`exp`、`iat`、`jti` 和角色 Claim；
- Token 失效不能只依赖自然过期，必须有会话/撤销联动策略。

## 13. 登出与会话撤销

支持以下操作：

- 当前浏览器会话登出；
- 按账号撤销全部活动会话；
- 按会话 ID 精确撤销；
- 密码重置后自动撤销旧会话；
- 账号禁用后自动撤销全部会话。

所有撤销动作都必须产生审计事件。

## 14. 账号锁定、禁用与重置密码

### 14.1 锁定

- 由连续失败登录触发；
- 支持自动解锁时间和管理员手动解锁；
- 锁定期间拒绝登录。

### 14.2 禁用

- 由管理员发起；
- 禁用后拒绝所有新登录；
- 现有活动会话立即撤销。

### 14.3 重置密码

- 仅管理员可发起；
- 发起后生成一次性受控重置票据或临时凭据；
- 账号进入 `PASSWORD_RESET_REQUIRED`；
- 完成改密前不允许进入正式操作会话。

## 15. 角色来源与授权衔接

- 首批角色来源以 `M01` 持久化角色授予为准；
- `M01` 在认证成功后生成统一 `OperatorIdentity.roles`；
- `M02` 继续通过 `PolicyDecisionService` 做动作到角色的决策；
- 角色变更后，`M01` 必须有能力让旧会话失效，避免授权漂移；
- `M02` 不拥有账号、角色授予和会话事实源。

## 16. 审计事件

至少覆盖：

- 登录成功；
- 登录失败；
- 账号锁定；
- 账号解锁；
- 账号禁用；
- 账号启用；
- 管理员发起重置密码；
- 用户完成改密；
- 会话创建；
- 会话撤销；
- 用户登出；
- 角色授予；
- 角色撤销。

事件最少包含：

- `eventId`
- `timestamp`
- `operator`
- `targetAccountId`
- `action`
- `result`
- `reason`
- `traceId`
- `requestId`
- `policyVersion`（如涉及授权）

## 17. 密钥管理要求

- 会话签名密钥、内建 OIDC 签名密钥、密码重置令牌密钥必须分离；
- 生产密钥必须来自受控密钥源或受管配置；
- 必须支持轮换与生效时间管理；
- 禁止在源码、示例生产配置、测试数据和日志中出现真实密钥；
- 本地联调和自动化测试可使用专用测试密钥，但必须与生产隔离。

## 18. 契约先行项

在进入实现前，至少新增或版本化以下契约：

- 浏览器登录请求与响应契约；
- 会话查询响应契约；
- 管理员重置密码命令契约；
- 首次登录改密命令契约；
- 身份与会话审计事件 Schema；
- `M01` 公开给 `bootstrap` 的应用接口与错误契约。

现有 `backend/contracts` 中尚无身份相关契约，本轮必须补齐。

## 19. 测试与发布门禁

必须覆盖：

- 账号状态与密码规则单元测试；
- 登录成功、失败、锁定、禁用、重置、改密流程测试；
- 会话撤销、过期、角色变更失效测试；
- 身份契约与错误契约测试；
- 授权拒绝与审计事件集成测试；
- 开发模式、本地联调模式、生产内建模式的配置隔离测试。

阻止发布的情形包括：

- 可绕过登录、锁定、禁用、会话撤销或审计；
- 密钥来源不受控；
- 角色变更后旧会话仍长期有效；
- 身份事实依赖 Redis 而非关系型事实源；
- 契约未版本化或缺少回滚路径。

## 20. 本轮是否进入编码

当前结论：本轮先落设计与计划，不直接编码。

原因：

- 身份相关版本化契约尚未建立；
- `M01` 正式模块结构尚未落地；
- `bootstrap` 目前仍承载过多安全集成职责；
- 先编码会放大跨模块耦合，容易把生产身份能力继续写进 `bootstrap` 或 `local-oidc` 路径。

进入编码前必须满足：

- 契约清单确定；
- ADR 接受；
- `project-plan` 纳入正式任务；
- `M01/M02/bootstrap` 边界被文档固定；
- 相关测试策略和回滚策略明确。

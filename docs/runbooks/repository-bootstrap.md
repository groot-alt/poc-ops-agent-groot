# 远程仓库初始化与保护规则

## 适用范围

本文用于首次创建远程 GitHub 仓库，以及配置 T005 要求的分支和 CI 门禁。

## 初始化步骤

1. 创建公司内部私有仓库，并将本地仓库推送到远程。
2. 确认默认分支为 `master`；如改用 `main`，同步修改相关文档。
3. 配置真实的平台负责人和安全负责人团队。
4. 根据真实团队创建 `.github/CODEOWNERS`，不得提交占位团队名称。

## 默认分支保护

对默认分支启用以下保护规则：

- 禁止直接推送和强制推送。
- 必须通过 Pull Request 合并。
- 至少需要一名代码所有者批准。
- 新提交到达后，旧批准自动失效。
- 必须解决全部评审对话。
- 必须通过 `仓库规范检查`、`Maven 构建与测试` 和 `密钥扫描`。
- 禁止管理员绕过生产发布和安全评审门禁。

## 验证

创建测试分支和 PR，确认：

- 不符合规范的分支名称被 CI 拒绝。
- Maven 构建、仓库检查、密钥扫描和制品上传任务可运行。
- 未通过检查或未获批准时无法合并。
- CI 生成的候选制品可以下载，并包含构建元数据。

# T006 测试与评测基线

## 目的

本基线定义 P1 只读诊断 MVP 的最小测试、评测和验收口径，避免后端、前端、Skill 和执行链路各自演进却没有统一质量门禁。

## 适用范围

- `backend/control-plane`
- `backend/execution-worker`
- `backend/contracts`
- `backend/skills`
- `frontend/operator-console`

## 测试分层

| 层级 | 目标 | 当前要求 |
|---|---|---|
| 单元测试 | 校验领域规则、值对象、异常映射和路由决策 | 新增核心逻辑必须覆盖 |
| 模块集成测试 | 验证控制面模块协作、配置加载和接口契约 | 每个新增 API 至少有成功与失败路径 |
| 契约测试 | 验证 API、事件、Skill Schema 和执行命令格式 | 契约变更必须带兼容性校验 |
| 安全测试 | 验证授权拒绝、输入校验、只读边界和错误泄露 | P1 必须覆盖拒绝路径 |
| 评测测试 | 验证只读 Skill 路由、计划生成和拒绝行为 | 每次模型/路由策略变更必须回归 |

## 开发工作流

任何需求进入开发前，至少完成以下动作：

1. 明确本次变更属于后端、前端、契约、Skill 还是评测样本。
2. 明确本次变更影响的是只读链路、拒绝链路还是通用框架能力。
3. 为本次变更指定验证方式：
   - 单元测试
   - 集成测试
   - 契约样本校验
   - 人工演示验证
4. 明确需要沉淀的验收证据文件或报告位置。

## P1 最低门禁

- `backend` Maven `verify` 通过。
- 控制面基础接口必须覆盖：
  - 健康检查
  - API 文档
  - 配置加载
  - 统一异常返回
- 授权拒绝和输入校验至少各有一条自动化测试。
- 新增 Skill 契约必须提供正例和反例样本。
- 任何只读 MVP 演示都必须附带测试与评测报告。

### T007 当前最低自动化门禁

- `/internal/healthz` 成功返回服务状态、版本号和模块清单。
- `/internal/modules` 成功返回模块清单和配置版本号。
- `/v3/api-docs` 成功返回 OpenAPI 文档。
- 非法参数返回结构化 `400`。
- 明确业务异常返回结构化 `400`。

## 初始评测集

### 路由评测

- 只读诊断查询应命中只读 Skill。
- 含写操作意图的请求应被拒绝或挂起审批。
- 缺少目标环境或关键参数的请求应返回结构化错误。

### 安全拒绝评测

- 未认证请求
- 无角色请求
- 越权访问请求
- 请求参数不符合契约

### 事件评测

- 工作流开始事件
- 工作流结束事件
- 失败事件
- 审计事件

## 交付证据矩阵

| 任务 | 必须证据 | 当前落点 |
|---|---|---|

| T007 控制面服务骨架 | 控制器、配置、异常处理、OpenAPI、自动化测试 | `backend/control-plane/bootstrap` |

## 执行方式

本地执行：

```powershell
Set-Location .\backend
.\mvnw.cmd -f .\pom.xml -B -ntp verify
Set-Location ..
```

CI 必须执行：

- Maven Verify
- 仓库规范检查
- 密钥扫描
- 制品归档

## 验收产物

- 测试结果
- 评测样本与结论
- 已知风险和豁免说明
- 发布或回滚影响说明

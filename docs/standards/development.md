# 研发规范

`AGENTS.md` 是权威的全局规范。本文件提供日常实现工作的检查清单。

## 任务就绪检查

- 已明确任务 ID 和模块 ID。
- 验收标准可验证。
- 已识别依赖和必要契约。
- 已明确安全分类和副作用等级。
- 已定义可观测性和审计要求。
- 已理解发布和回滚影响。

## PR 检查清单

- 变更范围聚焦于一个完整事项。
- 遵守模块边界。
- 契约变更已经版本化并测试。
- 已包含正常和异常测试。
- 已测试授权拒绝和失败路径。
- 已按要求添加日志、指标、追踪和审计事件。
- 未提交密钥或敏感测试数据。
- 已更新文档、ADR 和运行手册。
- 已说明发布和回滚方式。

## 副作用分类

| 分类 | 含义 | 最低控制要求 |
|---|---|---|
| 只读 | 不修改目标系统状态 | 身份、策略、审计、超时 |
| 低风险变更 | 范围受限且可回滚的修改 | 持久化工作流、必要审批、幂等、回滚 |
| 高风险变更 | 范围广、权限高、破坏性强或难以回滚 | 强隔离、明确审批、独立评审、恢复演练 |

## 契约优先工作流

1. 定义或更新版本化契约。
2. 添加兼容性和校验测试。
3. 实现提供方和消费方变更。
4. 验证失败和降级行为。
5. 记录发布和回滚方式。

## 触发安全评审的情况

- 新增目标系统凭据或权限。
- 新增执行能力或网络目标。
- 新增生产副作用操作。
- 修改授权或审批行为。
- 修改 Skill 风险分类。
- 修改隔离或沙箱行为。
- 在 Prompt、事件、日志或制品中加入敏感数据。

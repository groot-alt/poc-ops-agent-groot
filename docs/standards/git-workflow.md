# Git 协作与分支规范

## 分支

- 默认保护分支为 `master`；迁移为 `main` 时同步修改 CI 和仓库保护规则。
- 开发分支使用 `feature/<task-id>-<summary>`。
- 缺陷修复使用 `fix/<task-id>-<summary>`。
- 文档和工程维护分别使用 `docs/`、`chore/` 前缀。
- Codex 创建的分支使用 `codex/` 前缀。

## 提交与评审

- Commit 标题使用英文祈使句，并聚焦一个完整变更。
- PR 必须填写任务 ID、模块 ID、风险、测试、发布和回滚方式。
- PR 合并前必须通过仓库规范检查、Maven Verify、密钥扫描和适用的安全扫描。
- 禁止直接向保护分支推送；至少需要一名代码所有者批准。
- M02、M05、M07、生产变更 Skill 和部署配置必须经过安全评审。

## 发布与回滚

- CI 只生成和归档候选制品，不直接操作生产环境。
- 发布流程必须使用不可变版本，并保留对应 Commit、测试结果和扫描结果。
- 回滚通过部署上一已验证版本完成；禁止通过在线修改代码或制品回滚。

# ADR 0005：无外部 IdP 场景下采用正式内建身份域与登录模块
- 状态：Accepted
- 日期：2026-06-07
- 负责人：架构负责人
- 相关模块：M01、M02、M11
- 相关任务：T011

## 背景

当前控制面已经具备以下安全相关基础：

- `dev-hs256` 与 OIDC 两类令牌认证模式；
- 标准浏览器登录入口、回调和退出流程；
- 基于 `OperatorIdentity` 的服务端 RBAC 与审计链路；
- 本地 `local-oidc` Mock OIDC 联调能力。

这些能力默认假设生产环境存在企业 SSO / 外部 IdP，或仅用于开发与本地联调。当生产环境没有企业 SSO / 外部 IdP 时，系统必须自己承担正式身份认证能力，包括账号、密码、会话、撤销和审计。

如果继续沿用当前做法，会出现两个问题：

1. 把 `local-oidc` 或浏览器登录集成代码误当成正式身份方案；
2. 把生产身份逻辑继续堆积在 `bootstrap` 集成层，破坏 `M01/M02` 模块边界。

## 决策

在“生产环境没有企业 SSO / 外部 IdP”的场景下，采用正式内建身份域与登录模块方案，具体如下：

1. 在 `M01` 内新增正式身份域，实现账号、密码凭据、浏览器会话、锁定、禁用、管理员受控重置密码和首次登录改密。
2. `M01` 生成统一 `OperatorIdentity`，继续复用现有 `M02` 授权与审计链路。
3. `M02` 只负责策略决策、风险拒绝和审计，不保存密码，不做登录校验，不拥有账号事实源。
4. `bootstrap` 仅负责 WebFlux / Spring Security 装配、HTTP 适配和运行模式切换。
5. `local-oidc` 明确保留为本地联调能力，不得升格为生产实现。
6. 首批不实现完整 MFA，只预留扩展口。
7. 首批不实现邮件自助找回密码，只支持管理员受控重置密码。

## 考虑过的备选方案

### 方案 A：把 `local-oidc` / Mock OIDC 升格为生产方案

未采纳原因：

- 当前实现明确只面向本地联调；
- 同进程、内存授权码、运行时临时密钥和固定默认账号不满足生产要求；
- 会把 Mock 语义错误地带入正式安全边界。

### 方案 B：只做表单登录，不做正式内建身份域

未采纳原因：

- 会把账号、密码、会话等生产事实散落在 `bootstrap` 或临时组件中；
- 与现有 OIDC / `OperatorIdentity` 统一链路割裂；
- 后续接入外部 IdP 时仍需重构。

### 方案 C：直接在 `M02` 中维护角色和账号状态

未采纳原因：

- 会让授权模块拥有身份事实源，破坏 `M01/M02` 分工；
- 容易形成跨模块读表和双重真相。

## 影响

### 正面影响

- 在无外部 IdP 的生产环境中具备正式身份能力；
- 保持统一 `OperatorIdentity` 和统一授权链路；
- 清晰分离身份认证与策略授权责任；
- 为未来外部 IdP 接入保留兼容路径。

### 负面影响

- 需要新增身份领域模型、持久化表、契约和测试；
- 需要额外的密钥管理和运维手册；
- 会拉长 `M01` 的首轮实施周期。

### 安全影响

- 正式引入密码和会话事实源后，密钥管理、审计和回滚要求更高；
- 必须确保账号状态变化能及时撤销旧会话；
- 必须保证生产模式与本地联调模式绝对隔离。

### 迁移影响

- 当前开发态 `dev-hs256` 和 `local-oidc` 继续保留；
- 新增生产内建身份模式，不直接替换现有联调模式；
- 后续可平滑增加外部 IdP 模式，不需要推翻 `M02`。

## 验证方式

- 新增身份契约与错误契约测试；
- 登录、锁定、禁用、重置、改密和会话撤销测试；
- 身份状态变化后的授权和审计集成测试；
- 三类运行模式的配置隔离测试；
- 安全评审覆盖密钥管理、日志脱敏和审计完整性。

## 发布与回滚

### 发布

1. 先提交规格、ADR、实现计划和契约设计。
2. 按 `M01` 正式模块结构实施。
3. 在开发环境完成单元、契约、集成和安全测试。
4. 通过受控配置启用生产内建身份模式。

### 回滚

- 如正式身份模式未通过验收，可继续使用现有开发态或外部 IdP 接入路径；
- 本地联调继续保留 `local-oidc`，但不得作为生产回退方案；
- 身份表结构和契约如有破坏性变化，必须提供版本化迁移与回滚脚本。

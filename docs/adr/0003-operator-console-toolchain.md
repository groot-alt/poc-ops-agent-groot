# ADR 0003：操作台前端工具链

- 状态：Accepted
- 日期：2026-06-06
- 负责人：前端负责人
- 相关模块：M09、M11
- 相关任务：P1 只读操作台

## 背景

P1 需要交付消费强类型语义事件的只读操作台。全局技术基线已确定 React 和 TypeScript，但要求在创建脚手架前通过 ADR 明确具体工具。

## 决策

操作台采用 React、TypeScript 严格模式和 Vite。首个切片仅通过控制面 API 和 SSE 读取诊断事件，不包含浏览器授权决策，也不直接访问 Worker 或目标系统。

## 考虑过的备选方案

### Next.js

当前操作台是公司内部单页应用，不需要服务端渲染或公开内容优化，因此暂不引入额外运行时和部署复杂度。

### 自定义 Webpack

可以满足要求，但需要维护更多构建配置，P1 没有足够收益。

## 影响

- 使用 TypeScript 严格类型校验外部语义事件。
- Vite 仅负责开发和静态构建，不承担授权逻辑。
- 后续引入测试框架、状态库或组件库前需要说明必要性。

## 验证方式

- `npm run build` 必须通过。
- 操作台不得出现 `any`。
- 只读诊断视图必须按语义事件类型渲染。

## 发布与回滚

构建生成静态制品。回滚通过部署上一已验证 Commit 的静态制品完成。

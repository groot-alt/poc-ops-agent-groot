# 设计范围追溯清单

本文件用于约束项目规划必须严格遵循《企业级智能运维 Agent 开放底座架构设计蓝图 V8.1》。

## 产品边界

- 系统为公司内部自研自用。
- 系统为单组织部署。
- 设计中没有租户或多租户概念。
- 不规划租户标识、租户隔离、租户路由、租户计费和租户管理。
- 用户权限边界使用设计中的用户、角色、Skill Category、参数权限、目标环境和审批机制表达。

## 设计直接定义的能力

| 设计能力 | 规划落点 |
|---|---|
| SSO/OAuth2/LDAP、JWT、RBAC | M01 接入网关与身份认证、M02 策略授权与审计 |
| 无企业 SSO / 外部 IdP 时的内建身份提供方、账号登录、会话撤销与受控重置密码 | M01 接入网关与身份认证，衔接 M02 策略授权与审计 |
| Skill 契约、Categories、RiskLevel、Executor、Interceptor、OutputType、Parameters | M03 Skill 契约与注册中心 |
| 多标签 Skill 路由 | M04 Agent 路由与模型交互 |
| ReAct、DAG、三级并行和黑板通信 | M05 工作流、M06 DAG 编排与制品黑板 |
| API 熔断与脚本受控重试 | M06 DAG 编排与容错 |
| Windows Worker、WDAC、Job Object、工作区 | M07 执行器与安全隔离 |
| 只读诊断和目标系统能力 | M08 运维 Skill 与目标系统集成 |
| 结构化语义事件、SSE、前端渲染和确权交互 | M09 操作台与语义事件流 |

| MySQL、Redis、本地工作区、审计和可观测性 | M10 可观测性与平台运维 |

## 为实现设计目标而补充的工程机制

以下内容不是新增产品能力，而是为了让设计中的安全、恢复、审计和生产运行目标能够落地：

- 持久化工作流作为执行事实源。
- 幂等键、补偿和人工接管。
- Skill 签名、灰度和回滚。
- 强类型 CommandEnvelope 和版本化跨模块契约。
- 受限 Worker、短期凭据和受控网络出口。
- 测试评测、安全验证、故障注入和发布门禁。
- 在没有外部 IdP 的部署中，由 M01 实现正式内建身份域、账号会话和受控密码重置，但不得改变单组织产品边界。
- Agent 工作会话用于组织讨论、结构化上下文和多个独立 workflow；它不替代工作流事实源，也不改变服务端授权与审计边界。

这些机制不得改变产品边界。需要新增产品能力时，必须先获得用户确认并更新设计。

## 禁止擅自引入的能力

- 多租户和商业化计费。
- 面向外部客户的开放平台能力。
- 未经审批的生产写操作。
- 任意脚本生成后直接执行。
- 由人格或 Prompt 降低组织安全要求。
- 与设计目标无关的技术平台化扩展。
- 把 `local-oidc` / Mock OIDC 直接当成生产身份方案。

## 规划变更检查

每次新增模块、任务、契约或基础设施前，回答：

1. 它对应 design 的哪一章节或目标？
2. 它是产品能力，还是实现该产品能力的工程机制？
3. 它是否改变自研自用、单组织部署的产品边界？
4. 它是否引入 design 未定义的新角色、新权限或新业务流程？
5. 如果改变产品边界，是否已经获得用户确认？

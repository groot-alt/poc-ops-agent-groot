# Agent 工作区需求规格

## 1. 目的与范围

Agent 工作区用于承载研发、DBA 与运维人员的持续排障主题，并将自然语言讨论、只读诊断任务、语义事件和审计证据组织在同一工作上下文中。

本规格属于 M04、M05、M09 的产品与交互需求。当前阶段仅设计 P1 只读诊断能力，不允许生产写操作、任意脚本执行、策略绕过或审计绕过。

文中的“工作会话”指 Agent 工作区中的排障主题，不等同于 M01 的浏览器登录会话。

## 2. 核心模型

- 一个工作会话承载一个持续工作主题，可以包含普通讨论和多个独立只读 workflow。
- 普通追问、补充上下文和结果解释不创建 workflow。
- 每次实际诊断执行必须创建独立 `workflowId`，并以结构化任务卡片展示。
- 同一工作会话允许多个 workflow 并发执行；切换会话后任务继续在后台执行。
- 工作流、事件、结果和审计事实来自服务端强类型数据，前端不得从展示文本推断执行或授权状态。

## 3. 会话生命周期

- 点击“新建会话”后立即进入空白草稿，不要求预先填写表单。
- 草稿在首次发送消息或添加结构化上下文时持久化，避免产生空会话。
- 会话支持重命名、固定和归档，不向普通操作员提供物理删除。
- 会话列表采用可收起侧滑栏，支持搜索、状态筛选、最近会话、运行中会话和固定会话。
- 重新打开历史会话时，先加载消息与任务快照，再对未终态 workflow 从最近 sequence 自动恢复。
- 恢复失败、事件缺口和重复事件必须在对应任务卡片内明确处理。

## 4. 空白会话与上下文

- 空白会话展示评审通过的场景模板、结构化上下文入口和示例问题，同时允许直接输入自由问题。
- P1 支持添加服务、环境、时间范围、告警、工单、历史 workflow 和脱敏文本。
- P1 不开放任意文件上传，避免提前引入制品扫描、脱敏、保留和访问控制复杂度。

## 5. 任务与自动执行

- Agent 识别到可执行意图后生成计划或任务卡片，卡片展示 Skill 版本、目标环境、参数摘要、策略状态、事件进度和结果摘要。
- 普通空白会话默认关闭自动执行。
- 只有逐个评审通过、绑定固定 READ_ONLY Skill、受限参数范围和目标范围的场景模板可预设自动执行。
- 自动执行仅省略操作员再次点击执行的动作，不得绕过服务端策略、Skill 风险校验、幂等、审计和 Worker 隔离。
- 任务卡片仅在后端声明支持时展示取消请求和受控重试。
- 受控重试创建新 attempt，并保留原失败记录、事件和审计证据。

## 6. 后台任务、通知与协作

- 用户切换会话后，执行中 workflow 继续运行。
- 会话列表和统一通知入口聚合运行中、失败、需接管和已完成状态。
- P1 支持同组织内只读分享；查看者必须明确接管会话后才能继续发消息或执行。
- 接管后的新操作绑定接管人的 operator 身份，并独立审计。
- P1 不支持多人同时编辑或同时执行同一会话。

## 7. 页面结构

- 顶部会话工具栏展示会话列表入口、当前会话标题、新建会话、上下文标签、自动模式、分享、接管和更多操作。
- 聊天主区同时展示普通消息气泡和结构化 workflow 任务卡片。
- 右侧面板具有两种状态：
  - 默认展示会话上下文、目标范围、自动模式和分享状态。
  - 点击任务卡片后展示所选 workflow 的策略、Skill、事件、结果和审计入口。
- 右侧任务详情必须明确绑定当前选中的 workflow，不得自动跟随最近任务跳动。
- 输入区提供结构化上下文入口和自由文本输入。

## 8. 后续最小领域对象

- `WorkConversation`：会话标识、标题、创建者、当前接管人、状态、上下文、自动模式和更新时间。
- `ConversationMessage`：消息标识、会话标识、operator、消息类型、时间和内容。
- `ConversationWorkflowRef`：会话标识、workflowId、状态、Skill、目标、attempt 和最近 sequence。
- `ConversationContextRef`：服务、环境、时间范围、告警、工单和历史 workflow 引用。

以上对象在进入真实实现前必须形成版本化契约并完成安全评审。本轮 HTML 原型改造不新增真实 API、持久化模型或授权逻辑。

## 9. 原型验收标准

- 页面不再使用“任务执行路径”说明卡片占据核心空间。
- 用户可以明确找到会话列表、新建会话、自动模式、分享和接管入口。
- 普通消息与 workflow 任务卡片在视觉和语义上清晰区分。
- 页面同时表达两个独立并发 workflow，且状态互不混淆。
- 右侧详情明确绑定选中的 workflow。
- 页面只表达 P1 READ_ONLY 能力，不出现生产写入、任意脚本或绕过策略的入口。

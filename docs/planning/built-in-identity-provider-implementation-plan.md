# 内建身份提供方与登录模块实现计划

> **给执行 Agent 的要求：** 按任务顺序逐项落地，优先补契约和模块骨架，再进入业务实现。每个任务都必须包含对应测试、审计和回滚说明。

**目标：** 在无企业 SSO / 外部 IdP 的前提下，为控制面建立生产可用的内建身份提供方与浏览器登录模块，并保持 `M01/M02/bootstrap` 边界清晰。

**架构：** 在 `M01` 新增正式身份域与应用接口，`bootstrap` 只做 WebFlux / Spring Security 装配和 HTTP 适配，`M02` 继续复用统一 `OperatorIdentity` 做策略与审计。

**技术栈：** Java 21、Spring Boot 3.2+、Spring Security WebFlux、R2DBC、JSON Schema / OpenAPI、现有审计与策略模块。

---

## 0. 当前实现状态

截至 2026-06-08，本计划首批验收范围已落地完成，已实现：

- 身份登录、会话查询、改密、管理员重置密码和错误响应契约；
- `M01` 正式账号、密码凭据、浏览器会话与锁定领域模型；
- `R2dbcAccountRepository`、`R2dbcPasswordCredentialRepository`、`R2dbcAccountSessionRepository`；
- `built-in` 模式下的登录、改密、会话撤销、管理员重置密码与首次改密闭环；
- `bootstrap` 与 Spring Security 的正式装配，且把阻塞型仓储调用隔离到 `boundedElastic`；
- 正式运行手册与回滚说明；
- 真实数据库集成测试与浏览器登录回归测试。

当前仍明确不在本计划验收范围内的内容：

- 完整 MFA 实装；
- 用户自助找回密码；
- 内建 OIDC Token 对外发行；
- 首个管理员账号自动开户工具。

## 1. 前置结论

当前不直接进入编码，先完成契约、模块骨架、迁移方案和测试设计。进入业务实现前，必须满足：

- 身份相关契约完成版本化草案；
- `M01` 正式模块目录和公共应用接口完成设计；
- 账号、会话、密码、重置和审计事件的数据模型固定；
- 运行模式和配置隔离方案明确；
- 安全评审关注点形成清单。

## 3. 任务拆分

### 任务 1：补齐身份契约与错误契约

目标：

- 为登录、会话、改密、管理员重置密码和身份错误响应建立版本化契约。

输出：

- 登录请求/响应 Schema
- 会话查询响应 Schema
- 改密命令 Schema
- 管理员重置密码命令 Schema
- 身份错误码约定
- 必要的 Java 值对象

风险：

- 如果先写实现再写契约，会把 `bootstrap` 耦合固化。

测试：

- `backend/contracts` 契约校验；
- Java 值对象与 Schema 对齐测试。

回滚：

- 若设计不稳，先保留为 `v1alpha` 或新增独立版本，不覆盖现有 API。

### 任务 2：扩展 M01 模块骨架

目标：

- 让 `modules/identity` 从“Claim 映射工具模块”扩展为正式身份模块。

输出：

- `api/`：模块公共应用接口
- `application/`：登录、改密、重置、会话查询用例
- `domain/`：账号、凭据、会话、重置票据、状态机
- `infrastructure/`：R2DBC 仓储、哈希器、会话存储、密钥提供者

风险：

- 若直接在现有 `bootstrap.security` 下堆代码，会破坏模块边界。

测试：

- 模块装配测试；
- 领域状态机单元测试。

回滚：

- 模块骨架可先以空实现或受控假实现接线，不切换生产路径。

### 任务 3：设计并落地身份持久化模型

目标：

- 以关系型数据库承载账号、角色授予、密码凭据、会话和重置票据事实源。

输出：

- 数据表设计
- 迁移脚本
- 索引与唯一约束
- 基础仓储接口

风险：

- 若把会话或失败计数放 Redis 作为唯一事实源，将违反安全基线。

测试：

- 迁移脚本测试；
- 仓储集成测试；
- 锁定与会话撤销一致性测试。

回滚：

- 所有迁移必须可逆或提供显式降级脚本；
- 如未启用正式模式，不影响现有路径。

### 任务 4：实现密码策略与凭据服务

目标：

- 提供强哈希、参数升级、弱密码拒绝和最近一次防复用能力。

输出：

- `PasswordHasher`
- `PasswordPolicy`
- `CredentialVerifier`
- `PasswordRotationService`

风险：

- 哈希参数固定写死且不可升级；
- 错误提示过细导致用户名枚举。

测试：

- 正确密码/错误密码测试；
- 弱密码拒绝测试；
- 参数升级测试；
- 最近一次密码复用拒绝测试。

回滚：

- 支持多版本哈希验证，避免一次性强制迁移所有存量凭据。

### 任务 5：实现登录、锁定与会话服务

目标：

- 建立正式内建登录流程、失败计数、锁定和服务端会话治理。

输出：

- `LoginService`
- `SessionService`
- `AccountLockoutPolicy`
- 会话 Cookie 与过期策略

风险：

- 会话固定攻击；
- 锁定状态和会话状态不同步；
- 角色变更后旧会话仍可用。

测试：

- 登录成功/失败测试；
- 锁定触发与解锁测试；
- 空闲过期与绝对过期测试；
- 角色撤销后的会话失效测试。

回滚：

- 正式登录入口通过配置开关启用；
- 未启用时继续保留现有路径。

### 任务 6：实现管理员受控重置密码与首次登录改密

目标：

- 实现首批受控重置密码闭环，不包含邮件自助找回。

输出：

- 管理员重置密码命令
- 一次性重置票据或临时凭据机制
- 首次登录改密流程
- 强制改密状态机

风险：

- 重置后旧会话未撤销；
- 改密前提前建立正式会话。

测试：

- 管理员重置成功测试；
- 旧会话撤销测试；
- 首次登录改密必经测试；
- 改密成功后恢复 `ACTIVE` 测试。

回滚：

- 该能力只在正式内建身份模式启用；
- 可通过禁用管理入口停止新增重置请求。

### 任务 7：接入 bootstrap 与 Spring Security

目标：

- 让 `bootstrap` 只作为适配层，复用 `M01` 应用接口完成 HTTP 与安全装配。

输出：

- 正式登录控制器
- 会话查询控制器
- 改密控制器
- 管理员重置控制器
- 新的 `AuthenticatedPrincipalOperatorIdentityResolver` 适配

风险：

- 控制器直接调用仓储；
- 安全逻辑散落在 `bootstrap`。

测试：

- WebFlux 集成测试；
- 未登录 `401`、角色不足 `403` 测试；
- 浏览器登录主路径测试。

回滚：

- 通过运行模式切换回开发态或外部 IdP 路径。

### 任务 8：完善审计事件与 M02 衔接

目标：

- 把身份生命周期事件纳入现有审计链路，并确保 `M02` 继续只消费统一角色。

输出：

- 身份审计事件模型
- 审计写入扩展
- 角色变更触发会话失效机制

风险：

- 账号状态变化未留审计；
- `M02` 直接读取 `M01` 表。

测试：

- 登录成功/失败审计测试；
- 会话撤销审计测试；
- 角色变更后的授权回归测试。

回滚：

- 审计事件追加式记录，不覆盖历史结构；
- 若新事件消费方未准备好，可先只写入后端。

### 任务 9：补齐运行模式、密钥管理与手册

目标：

- 把 `dev`、`local-oidc`、`prod-built-in-idp` 三种模式清晰隔离。

输出：

- 新配置项与说明
- 生产密钥来源规范
- 运行手册与回滚手册

风险：

- 生产误启 `local-oidc`；
- 密钥落入仓库或示例环境变量。

测试：

- 配置装配测试；
- 模式互斥测试；
- 文档校验与手工演练。

回滚：

- 运行模式切换通过配置完成；
- 手册中必须写明关闭正式模式后的降级路径。

## 4. 依赖顺序

建议顺序：

1. 契约与错误码
2. `M01` 正式模块骨架
3. 数据模型与迁移脚本
4. 密码与凭据服务
5. 登录、锁定与会话服务
6. 管理员重置密码与首次登录改密
7. `bootstrap` 安全接线
8. 审计与 `M02` 回归
9. 运行手册、回滚手册和模式校验

禁止跳过 1 到 3 直接写控制器。

## 5. 风险点

- 把生产身份能力继续写进 `bootstrap` 或 `local-oidc`；
- 账号、角色、会话事实源分散，出现双重真相；
- 登录状态变化未同步到会话撤销；
- 密钥治理和日志脱敏被遗漏；
- 过早加入 MFA、邮件找回等扩展，导致超出 P1 范围；
- 契约未版本化，后续无法平滑演进。

### 6.3 集成测试

- 登录成功/失败；
- 锁定/禁用；
- 管理员重置密码；
- 首次登录改密；
- 会话失效；
- 角色变更后的授权拒绝；
- 审计事件写入。

### 6.4 安全测试

- 用户名枚举防护；
- 会话固定攻击防护；
- 密码与密钥不落日志；
- 模式隔离；
- `local-oidc` 禁止进入生产配置。

## 7. 回滚方案

- 通过运行模式配置回退到开发态或未来外部 IdP 模式；
- 正式身份相关数据库迁移必须提供回滚脚本或降级步骤；
- 会话与账号表上线前先在隔离环境完成演练；
- 生产启用前保留现有只读诊断能力，不与身份实施绑定上线；
- 如正式登录路径出现缺陷，优先关闭生产内建身份入口，不得启用 `local-oidc` 作为生产回退。

## 8. 当前是否具备进入编码的前提

当前结论：仍然不具备直接进入编码的前提。

原因：

- 身份相关版本化契约尚未落仓；
- `M01` 正式模块分层还没有骨架；
- 数据模型与迁移策略尚未审查；
- 安全关键模块 `M02` 相关联动仍需明确测试门禁。

满足以下条件后才建议进入编码：

- 契约草案完成并过审；
- `M01` 模块分层和命名冻结；
- 数据迁移与回滚方案确定；
- 安全测试清单可执行；
- 相关任务已纳入 `project-plan`。
